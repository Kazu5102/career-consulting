# システム開発原則：自己完結型データ連携システム

このプロジェクトは、以下の特許コンセプトに基づき開発・運用される。

## 1. データ管理の基本原則（ゼロトラスト）
- **永続的DB保存の回避:** ユーザーの対話履歴は、システム提供者側のデータベースに永続保存してはならない。
- **揮発性管理:** 対話データはブラウザの揮発性メモリ（State）上でのみ管理し、セッション終了後は消去すること。

## 2. 自己完結型HTMLパッケージ
- **エクスポート形式:** データ保存時は、暗号化されたデータと、それを復号するためのJavaScriptプログラムを内包した「スタンドアロン型HTMLファイル」として出力する。
- **暗号化:** ユーザーが設定したパスワードから、Web Crypto API等を用いて強力な鍵導出を行い、AES-GCM等で暗号化すること。

## 3. 専門家へのセキュア連携
- **パスワード必須:** 専門家（管理者）がデータを確認するには、物理的または口頭で共有されたユーザー独自のパスワードが必須となる。
- **非永続展開:** 復号されたデータは管理者端末のメモリ上にのみ展開し、管理者側のDBにも保存してはならない。

## 4. 動的介入（タイピング・フルーエンシー）
- **入力ゆらぎの活用:** ユーザーの打鍵間隔（ミリ秒）から「推敲係数（迷いの度合い）」を算出し、AIへのプロンプトに「非言語的コンテキスト」として付加することで、より共感度の高い介入を行う。

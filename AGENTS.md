# システム開発原則：自己完結型データ連携システム

このプロジェクトは、以下の特許コンセプトに基づき開発・運用される。

## 1. データ管理の基本原則（ゼロトラスト）
- **永続的DB保存の回避:** ユーザーの対話履歴は、システム提供者側のデータベースに永続保存してはならない。
- **揮発性管理:** 対話データはブラウザの揮発性メモリ（State）上でのみ管理し、セッション終了後は消去すること。

## 2. 自己完結型HTMLパッケージ
- **エクスポート形式:** データ保存時は、暗号化されたデータと、それを復号するためのJavaScriptプログラムを内包した「スタンドアロン型HTMLファイル」として出力する。
- **暗号化:** ユーザーが設定したパスワードから、Web Crypto API等を用いて強力な鍵導出を行い、AES-GCM等で暗号化すること。

## 3. 専門家へのセキュア連携
- **パスワード必須:** 専門家（管理者）がデータを確認するには、物理的または口頭で共有されたユーザー独自のパスワードが必須となる。
- **非永続展開:** 復号されたデータは管理者端末のメモリ上にのみ展開し、管理者側のDBにも保存してはならない。

## 4. 動的介入（タイピング・フルーエンシー）
- **入力ゆらぎの活用:** ユーザーの打鍵間隔（ミリ秒）から「推敲係数（迷いの度合い）」を算出し、AIへのプロンプトに「非言語的コンテキスト」として付加することで、より共感度の高い介入を行う。

## 5. 詳細仕様書（SYSTEM_SPECIFICATION.md）の遵守と更新管理ルール（案A）
- **仕様書の最優先参照**: AIは開発・改修を開始する前に、必ずプロジェクトルートの `SYSTEM_SPECIFICATION.md` を全行読み込み、そこに記述された詳細な機能・アルゴリズム・描画仕様を厳格に再現・遵守しなければならない。
- **仕様書と実装の同期更新**: コードに機能追加や仕様変更を加えた際は、必ず `SYSTEM_SPECIFICATION.md` の該当セクションも同時に最新の状態へアップデートすること。
- **バージョン一貫性の監査**: AIは、以下のバージョン定義がすべて【完全に一致していること】を監査し、最新版の適用状況を判断する。不一致がある場合は、コードや仕様書の反映漏れとみなし、即座に修正・同期しなければならない。
  1. `SYSTEM_SPECIFICATION.md` 内のバージョン表記（例: `Version 6.59`）
  2. `metadata.json` の `"version"` フィールド（例: `"6.59"`, アプリ名末尾も `Career Consulting_6.59` とする）
  3. 各プログラムソース（Reactコンポーネント、ビュー、サービス等）の冒頭コメントに記述されたバージョン（例: `// components/AIAvatar.tsx - v6.59 - 2026-06-30 - ...`）
  4. 主要コード内部 of バージョン定数（例: `const VERSION = "6.59"` などの文字列定義）
